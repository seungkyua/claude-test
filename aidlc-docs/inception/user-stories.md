## EPIC-001: 행사 정보 열람

### US-001: 행사 핵심 정보 확인

```
As a 잠재 참가자,
I want to 홈페이지 진입 시 행사명, 날짜, 장소를 즉시 확인하고 싶다,
So that 내가 참석 가능한지 빠르게 판단할 수 있다.
```

수용 기준:
- [ ] Given 홈페이지 접속 시, When 페이지 로드 완료, Then 행사명/날짜/장소가 히어로 섹션 최상단에 표시된다
- [ ] Given 히어로 섹션 확인 시, When CTA 버튼 클릭, Then 등록 폼으로 스크롤 이동한다

### US-002: 행사 소개 내용 파악

```
As a 잠재 참가자,
I want to KCD Seoul이 어떤 행사인지 이해하고 싶다,
So that 참가 여부를 결정할 수 있다.
```

수용 기준:
- [ ] Given 행사 소개 섹션 확인 시, When 스크롤 다운, Then 행사 목적/대상/주요 주제가 명확히 표시된다

## EPIC-002: 발표자 및 세션 탐색

### US-003: 발표자 목록 확인

```
As a 참가 희망자,
I want to 발표자 목록과 프로필을 보고 싶다,
So that 어떤 전문가들이 발표하는지 알 수 있다.
```

수용 기준:
- [ ] Given 발표자 섹션 확인 시, When 카드 렌더링, Then 사진/이름/소속/발표 주제가 표시된다
- [ ] Given 모바일 환경에서, When 발표자 섹션 확인, Then 1~2열 그리드로 반응형 표시된다

### US-004: 세션 일정 확인

```
As a 참가 확정자,
I want to 세션 일정을 타임라인으로 보고 싶다,
So that 어떤 세션에 참석할지 미리 계획할 수 있다.
```

수용 기준:
- [ ] Given 일정 섹션 확인 시, When 렌더링, Then 시간/트랙/발표자/주제가 시간 순으로 표시된다
- [ ] Given 복수 트랙 존재 시, When 트랙 전환, Then 각 트랙의 세션이 구분 표시된다

## EPIC-003: 참가 등록

### US-005: 참가 등록 폼 제출

```
As a 참가 희망자,
I want to 홈페이지에서 바로 참가 신청을 하고 싶다,
So that 별도 사이트 이동 없이 등록을 완료할 수 있다.
```

수용 기준:
- [ ] Given 등록 폼 접근 시, When 이름/이메일/소속/직책 입력, Then 필수 필드 유효성 검사가 실행된다
- [ ] Given 유효한 정보 입력 시, When 제출 버튼 클릭, Then 성공 메시지가 표시된다
- [ ] Given 잘못된 이메일 형식 입력 시, When 제출 시도, Then 에러 메시지가 해당 필드 하단에 표시된다

## EPIC-004: 스폰서 노출

### US-006: 스폰서 정보 확인

```
As a 스폰서사 담당자,
I want to 우리 회사 로고가 등급에 맞게 표시되기를 원한다,
So that 스폰서십 가치를 증명할 수 있다.
```

수용 기준:
- [ ] Given 스폰서 섹션 확인 시, When 렌더링, Then Gold/Silver/Bronze 등급별로 로고 크기가 차등 표시된다
- [ ] Given 로고 클릭 시, When 클릭, Then 스폰서사 웹사이트가 새 탭으로 열린다
